RESTful services give the flexibility to
create application involving the following operations(also known as CRUD).

  • Create
  • Read/Retrieve
  • Update
  • Delete

While performing these operations in RESTful services, there are guidelines or
principles on using HTTP methods for a certain type of operation.

Below is the list of method that is been used in creating a RESTful API:

  • GET
  • POST
  • PUT
  • DELETE
  • PATCH

GET METHOD:

Http get method is used to retrieve information from the REST API. GET is idempotent, meaning that regardless of how many times it repeats with the same parameters the results are the same.

POST METHOD:

In the REST API is only used to create a new resource. In some cases post method is used to update entity but this operation is not very frequent. POST API is not idempotent nor safe and should be used with care.

PUT METHOD: Put method is used to update an existing resource. Put method has some features as compare to the post method e.g (i) Put method is idempotent; this means the client can send the same request multiple times and as per HTTP spec, this has exactly the same effects as sending once. (ii) Put method is just like a putting in resource; This means completely replacing whatever is available at the given URL with the given data.

DELETE METHOD: This method is used to remove a given resource. Similar to PUT method, DELETE operation is idempotent.

e) PATCH METHOD: Partial update to a resource happens through PATCH METHOD.The main difference between PUT and PATCH in REST API is that PUT handles updates by replacing the entire entity, while PATCH only updates the fields that you give it.
